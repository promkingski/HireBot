Reference Architecture

How the Mission Log Analyzer is structured on Google Cloud Platform

🌐
Cloud Run
Frontend + API
Containerized FastAPI serving both the UI and REST endpoints. Auto-scales to zero, pay-per-request. Hosts the analyzer, chatbot, and admin interface.
🪣
Cloud Storage
File + Knowledge Store
Regional bucket with IAM-scoped access. Uploaded mission logs and chatbot knowledge documents are stored here.
🧠
Gemini 2.5 Flash
AI Engine (Vertex AI)
Powers both the document summarization and the career chatbot. Context-loaded with knowledge documents for personalized responses.
🔐
Secret Manager
Credential Vault
API keys and service credentials injected as env vars at deploy time. No secrets in code or containers.
📊
Cloud Logging
Observability
Request logs, latencies, errors. Optional BigQuery export via Log Sinks for longer-term analytics.
🏗️
Cloud Build
CI/CD Pipeline
GitHub-connected trigger. Every push to main rebuilds the container, pushes to Artifact Registry, and redeploys to Cloud Run.
Data Flow
User → Cloud Run (FastAPI) → Cloud Storage → Gemini 2.5 Flash → JSON Response
        └── Cloud Logging → Cloud Monitoring → BigQuery (optional)
